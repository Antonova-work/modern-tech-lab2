## Вопрос 1

Что выведет этот код? Объясните порядок выполнения блоков.

```java
class Demo {
    static { System.out.print("A "); }
    { System.out.print("B "); }
    Demo() { System.out.print("C "); }
    public static void main(String[] args) {
        System.out.print("D ");
        new Demo();
        System.out.print("E ");
        new Demo();
    }
}
```

**Предсказанный вывод:**

```text
A D B C E B C
```

**Объяснение**:
1. Сначала загружается класс Demo, поэтому срабатывает static блок (A).

2. Выполняется первая строка main (D).

3. При создании new Demo() сначала отрабатывает блок инициализации экземпляра (B), а затем конструктор (C).

4. Печатается (E).

5. При создании второго объекта new Demo() снова срабатывают блоки (B) и (C). Статический блок больше не вызывается.

## Вопрос 2

Класс `Child` в пакете `b`, класс `Parent` в пакете `a`. `Child extends Parent`. Какие члены `Parent` доступны в `Child`? А какие доступны в другом классе пакета `b`, который НЕ наследует `Parent`?

**Ваш ответ:** В классе Child доступны члены с модификаторами public и protected. В обычном классе пакета b (не наследнике) доступны только члены с модификатором public. Модификаторы private и default (package-private) в обоих случаях из другого пакета недоступны.

## Вопрос 3

Можно ли создать sealed-класс, у которого все наследники — record? Какое преимущество это даёт при использовании в switch?

**Ваш ответ:** Да, это возможно. Основное преимущество — исчерпываемость. При использовании switch компилятор проверяет, все ли возможные реализации (рекорды) обработаны. Если да, то блок default становится необязательным, а добавление нового наследника в будущем вызовет ошибку компиляции в switch, напоминая о необходимости его обработки.

## Вопрос 4

Что выведет этот код? Объясните, что такое String Pool и как работает `intern()`.

```java
String a = "Hello";
String b = new String("Hello");
String c = b.intern();
System.out.println(a == b);
System.out.println(a == c);
System.out.println(b == c);
```

**Предсказанный вывод:**

```textmate
false
true
false
```

**Объяснение:**
String Pool — это специальная область памяти для хранения уникальных строк.

1. a == b — false, так как b создан через new в обычной куче (heap), а a — в пуле.

2. a == c — true, так как метод intern() ищет строку в пуле и возвращает ссылку на уже существующую там строку "Hello".

3. b == c — false, это разные объекты (один в heap, другой в пуле).

## Вопрос 5

Почему этот код не компилируется? Как исправить?

```java
interface A { default void hello() { System.out.println("A"); } }
interface B { default void hello() { System.out.println("B"); } }
class C implements A, B { }
```

**Ваш ответ:** Код не компилируется из-за конфликта имен: класс C наследует два метода hello() с реализацией по умолчанию из разных интерфейсов, и возникает неопределенность.
Как исправить: Нужно переопределить метод hello() в классе C и либо написать свою логику, либо явно вызвать метод одного из интерфейсов: A.super.hello();.

## Вопрос 6

В чём разница между `StringBuilder` и `StringBuffer`? Приведите конкретный сценарий, когда нужен именно `StringBuffer`.

**Ваш ответ:** Разница в потокобезопасности. StringBuffer синхронизирован (thread-safe), а StringBuilder — нет. StringBuilder быстрее и предпочтительнее для работы в одном потоке. StringBuffer необходим, когда несколько потоков одновременно модифицируют одну и ту же строку (например, общий логгер или разделяемый буфер данных).

## Вопрос 7

Что выведет этот код? Объясните, что происходит с переменной `x` в лямбде.

```java
int x = 10;
Runnable r = () -> System.out.println(x);
// x = 20;  // что будет, если раскомментировать?
r.run();
```

**Предсказанный вывод:**

```textmate
10
```

**Объяснение:**
Лямбда-выражения могут использовать переменные из внешнего контекста, только если они являются final или effectively final (не меняются после инициализации). Если раскомментировать x = 20, код не скомпилируется, так как переменная перестанет быть effectively final.

## Вопрос 8

Напишите пример лямбда-выражения, которое нельзя заменить на ссылку на метод. Объясните почему.

**Ваш пример и объяснение:**

Пример: list.forEach(s -> System.out.println(s + "!")).

Здесь нельзя использовать System.out::println, так как в лямбде происходит дополнительная операция — конкатенация со строкой (+ "!"). Ссылка на метод возможна только тогда, когда лямбда просто передает свои аргументы в метод без изменений.

## Вопрос 9

Чем `EnumMap` быстрее `HashMap` для ключей-enum? Опишите внутреннюю реализацию в одном предложении.

**Ваш ответ:** EnumMap быстрее, потому что внутри она реализована как простой массив, где порядковый номер константы (ordinal()) служит индексом, что исключает вычисление хэш-кодов и разрешение коллизий.

## Вопрос 10

Дан `record Point(int x, int y) {}`. Что сгенерирует компилятор автоматически? Можно ли добавить в record изменяемое поле? Можно ли наследоваться от record?

**Ваш ответ:** Компилятор создаст: final поля, конструктор, геттеры (x() и y()), методы equals(), hashCode() и toString(). Добавить изменяемое (не-final) поле нельзя, так как рекорды по определению неизменяемы. Наследоваться от рекорда нельзя, так как они неявно являются final классами.
