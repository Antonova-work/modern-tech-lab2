# Задание 2.4 — Абстрактный класс vs Интерфейс

## Теоретическое напоминание

| Критерий | Абстрактный класс | Интерфейс |
|----------|-------------------|-----------|
| Отношение | «является» (is-a) | «умеет» (can-do) |
| Наследование | Одиночное (`extends`) | Множественное (`implements`) |
| Поля (состояние) | Могут быть любые | Только `static final` |
| Конструктор | Есть | Нет |
| Методы с реализацией | Да | `default`/`static` (с Java 8) |

**Правило выбора:**
- Если есть общие **поля** (состояние) и отношение «является» → **абстрактный класс**.
- Если нужна **множественная реализация** или отношение «умеет» → **интерфейс**.

## Ваша таблица

| Ситуация | Ваш выбор | Обоснование |
|----------|-----------|-------------|
| Все животные имеют имя и возраст, но звучат по-разному | Абстрактный класс | Отношение «является» (is-a). Есть общие поля состояния (имя, возраст). |
| Некоторые объекты (документ, изображение, настройки) можно сериализовать | Интерфейс | Отношение «умеет» (can-do). Объекты разных типов объединяются общей способностью. |
| Все транспортные средства имеют скорость и топливо, но двигаются по-разному | Абстрактный класс | Отношение «является». Имеются общие поля (скорость, топливо) и базовая логика. |
| Робот, посудомоечная машина и телефон — все умеют подключаться к Wi-Fi | Интерфейс | Отношение «умеет». Разнородные объекты реализуют общую функциональность. |
| Круг, квадрат и треугольник — все фигуры с площадью и периметром, у всех есть цвет | Абстрактный класс | Отношение «является». Общее состояние (цвет) и методы (площадь, периметр). |
| Класс должен одновременно уметь летать, плавать и ездить | Интерфейс | Требуется множественное наследование поведения, которое доступно только через интерфейсы. |
